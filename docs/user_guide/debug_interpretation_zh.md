# 调试可视化指南 (Debug Visualization Guide)

本指南旨在说明如何解读 PineappleHub 调试模式下显示的中间步骤图像。理解这些输出对于诊断光照、对焦或算法参数问题至关重要。

---

## 1. 平滑处理 (Smoothing)
**含义：** 应用高斯模糊 ($\sigma=1.0$) 后的原始图像。
*   **正常：** 略微模糊，但果眼的边缘仍然清晰可见。传感器噪点被抑制。
*   **调试：** 如果图像太模糊，可能是原始照片**失焦**。如果噪点太多，可能是相机的 ISO 设置过高。

## 2. 尺度校准 (Scale Calibration)
**含义：** 在图像中自动定位物理参照物（硬币，半径 12.5 mm），推算像素/毫米比例尺 $S$（单位：px/mm）。

**算法流程（`perform_scale_calibration`）：**
1.  **轮廓提取（`extract_robust_contours`）：** 对灰度图像依次执行大津二值化、形态学闭运算（$r=2$，L2 范数）、形态学开运算（$r=3$，L2 范数），提取外轮廓，并通过 `remove_hypotenuse` 滤除直线边缘（直尺等干扰）。
2.  **凸包修复：** 对每个面积 $> 100$ px 的候选轮廓，计算其**凸包**（Graham 扫描算法），抹平污渍或遮挡导致的边缘凹陷，从而得到更准确的形状描述。
3.  **几何特征计算（旋转不变量）：**
    *   **纵横比** $ar = d_{\min} / d_{\max}$：其中 $d_{\min}$、$d_{\max}$ 为最小外接矩形（`min_area_rect`）的短边与长边，范围 $(0, 1]$，圆形理论值为 $1.0$。**注意：不使用轴对齐包围盒，以避免旋转角度对宽高比的扭曲。**
    *   **填充率** $\phi = A_{hull} / (d_0 \cdot d_1)$：凸包面积与最小外接矩形面积之比，圆形理论值约 $\pi/4 \approx 0.785$。
    *   **圆形度** $C = 4\pi A_{hull} / P_{hull}^2$：其中 $P_{hull}$ 为凸包周长，圆形理论值为 $1.0$。
4.  **两级回退检测：**
    *   **Tier 1（严格）：** $ar > 0.95$、$\phi \in (0.70, 0.88)$、$C > 0.85$，在通过条件的候选中选凸包面积最大者。
    *   **Tier 2（宽松+打分，仅在 Tier 1 无结果时触发）：** $ar > 0.85$、$\phi \in (0.60, 0.92)$、$C > 0.70$，在通过宽松门控的候选中按得分 $score = -(10|ar-1| + 5|\phi - \pi/4| + 5|C-1|)$ 取最高分者。
5.  **尺度推算：** 由凸包面积反推硬币像素半径 $r_{px} = \sqrt{A_{hull}/\pi}$，从而得 $S = r_{px} / 12.5$。

*   **正常：** 硬币被红色矩形框和红圈标注。
*   **调试：** 如果未检测到硬币（没有红色标记），查看浏览器控制台的 `[CoinCandidate]` 日志，检查各指标是否接近阈值。若出现 `Tier 1 failed` 提示则可进一步查看 `[CoinDetect T2]` 的打分详情。常见原因：硬币被大面积遮挡（边缘损失超过约 30%）、强反光导致内部割裂、或与背景对比度不足。

## 3. 纹理块/二值化 (Texture Patch / Binary)
**含义：** 应用大津（Otsu）全局阈值后的二值图像。
*   **正常：** 呈现为一张噪点较多的"星图"。果眼中心显示为白色斑块，间隙显示为黑色。
*   **调试：**
    *   **全白**：对比度过低，光照可能过于均匀。
    *   **全黑**：对比度过高，或图像曝光不足。

## 4. 二值融合 (Binary / Morphological Closing)
**含义：** 步骤 3 的"星图"经过闭运算（$r=2$）和开运算（$r=3$）处理。细小的噪点合并为实心的斑块。
*   **正常：** 明显的、大致呈圆形的斑块，代表单个果眼。
*   **调试：**
    *   **连成一片**：形态学半径 $R_{morph}$ 过大（硬币检测可能有误，导致比例尺偏大）。
    *   **过度破碎**：形态学半径 $R_{morph}$ 过小。

## 5. 形态学/ROI提取 (Morphology / ROI Extraction)
**含义：** 最终裁剪出的"果皮区域"（ROI）三联图，从左到右依次为：
1.  **垂直展开图（Vertical Unwrap）**：将果实沿垂直轴的圆柱投影展开。
2.  **旋转校正 ROI（Rotated ROI）**：旋转扶正后的原始高分辨率裁剪图，菠萝主轴应垂直。
3.  **水平展开图（Horizontal Unwrap）**：将果实旋转 90° 后再展开，用于提取垂直方向（果实宽度）的度量。

**ROI 选取算法（两阶段）：**
1.  **识别**哪个候选是果皮侧：使用 Otsu 轮廓上的纹理评分（边缘密度 × √面积）。
2.  **定界**完整果实轮廓：对平滑灰度图以低阈值（$\tau = 25$）二值化，在质心处匹配目标轮廓。这避免了 Otsu 二值化本身导致的轮廓碎片化问题。

红色标注线（实线为主轴两端，虚线为中心线）标出了用于计算尺寸的最小外接矩形。

*   **正常：**
    *   菠萝主轴应垂直。
    *   图像应紧凑地包含大部分果眼区域。
    *   红色标注线应紧贴果实轮廓。
*   **调试：**
    *   **未扶正**：如果图像依然倾斜，说明 `MinAreaRect` 计算的角度有误。
    *   **空白**：如果裁剪区域完全错误，说明 ROI 评分失败（可能视场内有其他面积更大的非果实物体）。
    *   **ROI 同时包含果皮和切面**：低阈值（$\tau=25$）将两个物体合并为同一轮廓。这通常发生在果皮与切面摆放距离过近（间距 < 5 mm）时。请增大两者间距后重试。
    *   **红线偏离**：展开图中的标注线未紧贴果实，可能导致尺寸计算偏差。
